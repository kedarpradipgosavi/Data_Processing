Section 02
Clean → Encode → Split → Train
Step 3 — Clean & Prepare
import numpy as np

# Drop rows with missing or negative target
df = dataset.dropna(subset=['selling_price'])
df = df[df['selling_price'] > 0]

# Extract numbers from string columns
def extract_num(series):
    return pd.to_numeric(
        series.astype(str).str.extract(r'([\d.]+)')[0], errors='coerce'
    )

df['mileage_num']   = extract_num(df['mileage'])
df['engine_num']    = extract_num(df['engine'])
df['max_power_num'] = extract_num(df['max_power'])

# Remove dirty owner values
valid_owners = ['First Owner', 'Second Owner',
                'Third Owner', 'Fourth & Above Owner', 'Test Drive Car']
df = df[df['owner'].isin(valid_owners)]
Step 4 — Identify Features & Target
features = ['year', 'km_driven', 'fuel', 'seller_type',
            'transmission', 'owner', 'mileage_num', 'engine_num',
            'max_power_num', 'seats']

X = df[features].dropna()
y = df.loc[X.index, 'selling_price']

# One-hot encode categorical columns
X_encoded = pd.get_dummies(X, drop_first=True)
print(X_encoded.shape)  # (7897, 16)
Step 5 — Train Test Split
from sklearn.model_selection import train_test_split

X_train, X_test, y_train, y_test = train_test_split(
    X_encoded, y, test_size=0.2, random_state=42
)

print(f"Train: {X_train.shape[0]} rows")  # 6317
print(f"Test:  {X_test.shape[0]} rows")   # 1580
Step 6 — Feature Importance with Random Forest
from sklearn.ensemble import RandomForestRegressor

model = RandomForestRegressor(n_estimators=100, random_state=42)
model.fit(X_train, y_train)

# Build importance table
fi = pd.DataFrame({
    'Feature':    X_encoded.columns,
    'Importance': model.feature_importances_
}).sort_values('Importance', ascending=False)

print(fi.to_string())

Key Takeaway

5 features → almost same accuracy as 16 features

Using only max_power_num, year, km_driven, seats, engine_num gives 96.42% accuracy vs 96.77% with all features. This means 11 of the 16 features contribute almost nothing — feature selection saves computation with no real loss.

Discussion Questions — Click to reveal answer
🚗
Which feature affects car price the most? Why does engine power matter more than fuel type?
Answer

max_power_num (engine power in bhp) is the top feature at 59.9% importance. Engine power directly reflects a car's performance tier — budget cars typically have 60–80 bhp while premium cars have 150+ bhp. Buyers and sellers both use performance as the primary pricing signal. Fuel type (Petrol/Diesel/CNG), by contrast, affects running costs but not the car's fundamental value tier — that's why it scores less than 1%.

📅
Why is year important? If two identical cars exist — one 2015, one 2022 — which costs more, and why?
Answer

The 2022 car costs more. Year captures depreciation — cars lose value over time due to wear, outdated technology, and expired warranties. A newer car has more remaining lifespan, current safety features, and lower repair risk. Year scores 18.9% importance because this depreciation pattern is consistent across all 8,128 cars in the dataset. The model learned: newer year → higher price, almost universally.

🛣️
Why does km_driven reduce price? What does high mileage tell a buyer about a car?
Answer

High km_driven signals more engine wear, more stress on components, and higher near-future maintenance costs. A car with 200,000 km is likely to need major servicing soon — clutch replacement, engine overhaul, suspension work. Buyers price this risk in. km_driven scores 11.9% importance because this relationship is consistent: more kilometers = more wear = lower price. It is essentially measuring how much "life" is left in the car.

⛽
Why did fuel type score very low importance? Is fuel type truly irrelevant to price?
Answer

Fuel type is not truly irrelevant — Diesel cars do tend to cost more than Petrol — but the model finds it less important because engine power already captures most of this signal. Diesel cars typically have higher bhp and torque, so once max_power is known, fuel type adds little extra information. This is called feature correlation — two features that carry overlapping information. The model effectively says: "If I already know the power output, knowing the fuel type doesn't tell me much more about the price."
