Section 01

Introduction to Data Collection

In data science, before any model is trained, any chart is plotted, or any insight is extracted, data must first be collected. This phase is often underestimated — yet it consumes the majority of a data scientist's time and has the greatest impact on the final result.

01.1 What is Data Collection?

Data collection is the systematic process of gathering raw information from various sources in a structured or unstructured form, to be later processed, cleaned, and analysed. It answers the fundamental question: where does the data come from?

📊

Structured Data

Rows and columns — databases, spreadsheets, CSV files. Easily queryable with SQL or Pandas.

📝

Unstructured Data

Text, images, audio, video. Requires parsing, NLP, or computer vision to extract meaning.

🔀

Semi-Structured

JSON, XML, HTML — has some organisation but doesn't conform to a rigid table format.

01.2 Why Does It Matter?

✅ Good Data Collection

  • Accurate, reproducible analysis
  • High-quality ML model performance
  • Trustworthy business decisions
  • Efficient pipelines & automation

❌ Poor Data Collection

  • Biased or incomplete datasets
  • Flawed model predictions
  • Wasted processing resources
  • Legal and ethical violations
Section 02

Understanding Data Sources

Data can originate from a wide variety of sources. Understanding each source type — and when to use it — is a foundational skill for every data scientist.

02.1 APIs (Application Programming Interfaces)

APIs are the cleanest way to collect data. A server exposes a structured endpoint, and you send an HTTP request to retrieve organised data — usually in JSON format.

📡 How an API Call Works

→

→

→

api_example.py 
import requests

# Fetch weather data from a public REST API
url = "https://api.openweathermap.org/data/2.5/weather"
params = {
    "q": "Mumbai",
    "appid": "YOUR_API_KEY",
    "units": "metric"
}

response = requests.get(url, params=params)

if response.status_code == 200:
    data = response.json()
    print(f"Temperature: {data['main']['temp']}°C")
    print(f"Condition: {data['weather'][0]['description']}")
else:
    print(f"Error: {response.status_code}")

Relational databases (MySQL, PostgreSQL, SQLite) and NoSQL databases (MongoDB) are the backbone of most production systems. Data scientists extract data from these using SQL queries or Python connectors.

02.3 Web Pages

When no API is available, web scraping lets you extract data directly from HTML. Used for news, product prices, job listings, social content, and more.

02.4 Files

CSV, JSON, Excel, Parquet, XML — local or cloud-based files are a daily part of the data scientist's workflow. Python's Pandas library makes these trivial to work with.

02.5 Sensors & Logs

IoT devices generate real-time streams of data: temperature readings, location coordinates, click events. Application logs track user behaviour and system errors. These require streaming or batch ingestion pipelines.

Section 03

Ethical Considerations

Data collection is not just a technical challenge — it's a moral and legal responsibility. Violating data ethics can result in legal penalties, harming individuals, or destroying public trust.

⚠️ Critical Rules
  • 🔴 Always review Terms of Service before scraping any website.
  • 🔴 Respect robots.txt — it signals which parts of a site should not be crawled.
  • 🔴 Never collect personal data without informed consent (GDPR, CCPA).
  • 🔴 Do not overload servers — add delays between requests (time.sleep()).
  • 🔴 Attribute your sources and respect licensing and copyright.
🛡️

GDPR

EU regulation requiring consent for personal data collection and the right to erasure.

🌐

CCPA

California law giving users control over what personal data is collected and sold.

🤖

robots.txt

A file at /robots.txt on every website that specifies what bots can and cannot crawl.

⚖️

Fair Use

Use data only for the stated purpose. Do not redistribute proprietary datasets.

When NOT to Scrape

Official API exists — use it instead Terms of Service prohibit scraping Content is behind login (private) Scraping would cause server harm Personal/sensitive data involved

Section 05

Web Scraping

Web scraping is the automated extraction of data from web pages. Rather than manually copying information, a scraper reads and parses the page's HTML to collect exactly what is needed.

05.1 Why Use Web Scraping?

🤖

Automation

Replace hours of manual copy-pasting with a script that runs in seconds.

🔎

No API Available

Access data that exists only on web pages with no official API endpoint.

📈

Dataset Building

Construct training datasets for ML models from publicly available web content.

📡

Monitoring

Track price changes, news updates, or website changes over time.

05.2 The Scraping Pipeline

01

Identify the Target

Choose the website and specific data you want. Check the ToS and robots.txt. Plan your extraction strategy.

02

Fetch the HTML

Use Python's requests library to send a GET request and download the raw HTML content of the page.

03

Parse the HTML

Pass the HTML to BeautifulSoup, which builds a navigable parse tree you can search and traverse.

04

Extract Target Elements

Use tag names, CSS classes, IDs, or attributes to locate the exact data you need from the parse tree.

05

Clean & Store

Strip whitespace, remove unwanted characters, and save the structured data to a CSV, JSON, or database.

Section 06

HTML & CSS Basics for Scraping

Before writing a scraper, you need to understand the structure of a web page. Every webpage is built with HTML — you can inspect any page's structure using your browser's Developer Tools (Right-click → Inspect).

Section 07

The requests Library

The requests library is the standard tool for sending HTTP requests in Python. It's the first step in any web scraping workflow — fetching the raw HTML of a page.

fetch_page.py 
import requests

# Step 1: Define the URL
url = "https://quotes.toscrape.com/"

# Step 2: Add headers to mimic a real browser
headers = {
    "User-Agent": "Mozilla/5.0 (Windows NT 10.0; Win64; x64)"
}

# Step 3: Send GET request (with error handling)
try:
    response = requests.get(url, headers=headers, timeout=10)
    response.raise_for_status()   # Raise error for 4xx/5xx codes

    print(f"Status: {response.status_code}")
    print(f"Content-Type: {response.headers['Content-Type']}")
    print(response.text[:500])     # Print first 500 characters of HTML

except requests.exceptions.HTTPError as e:
    print(f"HTTP Error: {e}")
except requests.exceptions.ConnectionError:
    print("Network connection failed.")
except requests.exceptions.Timeout:
    print("Request timed out.")

07.1 Key Response Properties

Property / MethodDescriptionExample
response.status_codeHTTP status (200 = OK, 404 = Not Found)200
response.textPage HTML as a Unicode string"<html>..."
response.json()Parse JSON response body into dict{"name": "Delhi"}
response.headersResponse metadata (Content-Type, etc.){"Content-Type": "text/html"}
response.raise_for_status()Raises exception on 4xx/5xx codes—
🧠 Best Practices
  • Always set a timeout to avoid hanging indefinitely.
  • Use User-Agent headers to avoid being blocked.
  • Add time.sleep(1) between requests to be a polite scraper.
  • Always handle exceptions — networks are unreliable.
Section 08

BeautifulSoup

Once you have the HTML, you need to extract specific pieces of data. BeautifulSoup turns raw HTML into a navigable tree structure, making it easy to find elements by tag, class, ID, or attribute.

scrape_quotes.py 
import requests
from bs4 import BeautifulSoup
import csv

url = "https://quotes.toscrape.com/"
response = requests.get(url)
soup = BeautifulSoup(response.text, "lxml")

# Method 1: find() — finds the FIRST matching element
first_quote = soup.find("span", class_="text")
print(first_quote.text)

# Method 2: find_all() — finds ALL matching elements
all_quotes = soup.find_all("span", class_="text")

# Method 3: select() — CSS selector syntax
authors = soup.select("small.author")

# Extracting attributes
links = soup.find_all("a")
for link in links[:5]:
    print(link.get("href"))   # Safe way to access attributes

# Save to CSV
with open("quotes.csv", "w", newline="", encoding="utf-8") as f:
    writer = csv.DictWriter(f, fieldnames=["quote", "author"])
    writer.writeheader()
    for q, a in zip(all_quotes, authors):
        writer.writerow({"quote": q.text, "author": a.text})

💡 Safe Element Access

Always check if an element exists before accessing it to avoid AttributeError:

title = soup.find("h1")
if title:
    print(title.text)
else:
    print("Title not found")
Section 09

Databases & SQL Data Collection

Databases are the primary storage for enterprise data. As a data scientist, you'll frequently connect to databases, run queries, and load results into Pandas DataFrames for analysis.

09.1 SQLite with Python

SQLite is a lightweight, file-based database built into Python's standard library — perfect for learning and small projects.

database_collection.py 
import sqlite3
import pandas as pd

# Connect to database (creates file if it doesn't exist)
conn = sqlite3.connect("students.db")
cursor = conn.cursor()

# Create a table
cursor.execute("""
    CREATE TABLE IF NOT EXISTS students (
        id INTEGER PRIMARY KEY,
        name TEXT NOT NULL,
        grade TEXT,
        score REAL
    )
""")

# Insert sample data
students = [
    (1, "Aarav Mehta",   "A", 91.5),
    (2, "Priya Sharma",  "B", 78.0),
    (3, "Rohan Kapoor",  "A", 88.3),
]
cursor.executemany("INSERT OR IGNORE INTO students VALUES (?,?,?,?)", students)
conn.commit()

# Query directly into a Pandas DataFrame ✨
df = pd.read_sql_query("SELECT * FROM students WHERE score > 80", conn)
print(df)

conn.close()

09.2 Common SQL Data Collection Queries

sql_queries.sql 
-- Select all records
SELECT * FROM students;

-- Filter with conditions
SELECT name, score FROM students WHERE grade = 'A' AND score > 85;

-- Aggregate: count and average per grade
SELECT grade, COUNT(*) AS total, AVG(score) AS avg_score
FROM students
GROUP BY grade
HAVING COUNT(*) > 1;

-- Join two tables
SELECT s.name, c.class_name
FROM students s
JOIN classes c ON s.class_id = c.id;
Section 10

File Handling for Data Collection

Many real-world datasets arrive as files: CSV exports from systems, JSON from APIs, Excel reports from stakeholders. Python and Pandas make reading and writing all these formats straightforward.

file_collection.py 
import pandas as pd
import json

### ── CSV ── ###
df = pd.read_csv("data.csv")
df.to_csv("output.csv", index=False)

### ── JSON ── ###
with open("data.json") as f:
    raw = json.load(f)                 # dict or list
df_json = pd.DataFrame(raw)

### ── Excel ── ###
df_xl = pd.read_excel("report.xlsx", sheet_name="Sales")
df_xl.to_excel("output.xlsx", index=False)

### ── Reading from a URL (CSV on web) ── ###
url = "https://raw.githubusercontent.com/datasets/iris/main/data/iris.csv"
df_remote = pd.read_csv(url)
print(df_remote.head())

Session Wrap-Up

📌 Key Takeaways

🌐

APIs First

Always check for an official API before scraping. APIs are stable, structured, and legally unambiguous.

🕸️

Scrape Responsibly

Check ToS, respect robots.txt, add delays, and never collect personal data without consent.

🧩

HTML is the Map

Understanding HTML structure and CSS selectors is the foundation of all web scraping work.

🗄️

SQL is Essential

pd.read_sql_query() bridges the gap between databases and your data science workflow.

🛠️

Right Tool, Right Job

APIs → requests. Static HTML → BeautifulSoup. Dynamic JS → Selenium. Files → Pandas.

♻️

Error Handling Always

Networks fail, pages change, elements disappear. Defensive coding keeps your pipeline alive.
