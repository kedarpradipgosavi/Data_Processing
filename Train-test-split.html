Train & Test Split

Why we split data, how to do it, and hands-on practice

Hour 1
Concepts
The Core Problem

Model Memorization

A model that trains and tests on the same data is like a student who memorizes an answer key — they score 100% on that test, but fail every new question. This is overfitting. We split data so we can measure how well the model generalizes to unseen examples.

🏋️

Training Data

The subset the model learns from. Parameters are tuned exclusively against this data.

🧪

Testing Data

Held-out data the model never sees during training. Used only to evaluate final performance.

📌

Overfitting

Model performs great on training data but poorly on new data — it memorized noise, not patterns.

🌍

Generalization

The model's ability to make accurate predictions on data it has never encountered before.

Practical — sklearn
Python — Import
from sklearn.model_selection import train_test_split
Python — Usage
# X = features, y = labels
X_train, X_test, y_train, y_test = train_test_split(
    X, y,
    test_size=0.2,         # 20% goes to test set
    random_state=42       # seed for reproducibility
)
Parameter Reference
Parameter Type Description
X, y array-like Features matrix and labels. Must have the same number of rows.
test_size float Fraction of data for the test set. 0.2 = 20%. Default is 0.25.
random_state int Seeds the random number generator so the same shuffle happens every run. Any integer works; 42 is conventional.
shuffle bool Whether to shuffle before splitting. Defaults to True — almost always leave it on.
stratify array-like Pass y here to preserve class proportions in both sets. Important for imbalanced datasets.
Why random_state matters

Reproducibility

Without a fixed random_state, every run shuffles the data differently — giving you different splits, different results, and results that can't be compared or verified by others. Setting it to any integer (e.g. 42) locks the shuffle so your experiment is repeatable by anyone who runs the same code.

Python — Exercise Starter
import pandas as pd
from sklearn.model_selection import train_test_split
from sklearn.preprocessing import StandardScaler

# 1. Load
df = pd.read_csv("your_dataset.csv")

# 2. EDA
print(df.shape)
print(df.isnull().sum())
print(df.describe())

# 3. Prepare features and target
X = df.drop("target", axis=1)
y = df["target"]

# 4. Scale
scaler = StandardScaler()
X_scaled = scaler.fit_transform(X)

# 5. Split
X_train, X_test, y_train, y_test = train_test_split(
    X_scaled, y, test_size=0.2, random_state=42
)

print(f"Train: {X_train.shape} | Test: {X_test.shape}")
